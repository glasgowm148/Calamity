Tweet Stream Downloader

For 2019 onward, the tweet collections will be provided through a downloader tool. This replaces the orginal downloader tool that was an option available in 2019.

Getting the Tweets (JSON)

For each event, we provide a stream of tweets collected during that event to categorize. There are two ways that participants can download the tweets for each event.

1) By tweet ID using the Twitter API: The first method involves downloading the list of tweet identifiers for each event from this page, and then using an external tool (such as twarc or twitter-tools) to download each tweet by ID via the Twitter API. You can get the tweet IDs for the 2018 test topics via the button below:

2) Stream download via TREC-IS-Client-v2.jar: Twitter allows the hosting of small datasets (less than 50k tweets) and the track organizers maintain a server with the event tweets which you can use to download a copy directly. To do so, you need to download the TREC-IS-Client-v2.jar file using the button below. This jar will attempt to connect to the central server and download a copy of the tweets for each event, for a particular dataset. There are two currently available datasets 'trecis2018-test' and 'trecis2018-train'. It will write one file per event (GZIP) in the current directory. The jar was compiled with Java 1.8. We do not guarantee that this service will always be available. An example download command is shown below:

java -jar TREC-IS-Client-v2.jar --noNewLinesInTweet trecis2018-test

If you use this method for maintaining the corpus, note that the server outputs tweets using its own format, which differs from Twitter's own slightly. If you omit the --noNewLinesInTweet flag, it will pretty print the json, from which it is easier to see the structure.
